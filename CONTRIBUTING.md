# Contributing to Agent Tool Intelligence

First — thank you. This project exists because MCP builders and AI developers care about tool quality.

## Ways to Contribute

### 🏷️ Embed a Badge (5 seconds)
The easiest way to support the project:
```markdown
[![Agent Tool Intel](https://agent-tool-intel-production.up.railway.app/badge/YOUR_ORG%2FYOUR_REPO)](https://agent-tool-intel-production.up.railway.app)
```
Every badge helps another builder discover the platform.

### 🐛 Report a Bug
Open an issue. Include:
- What you expected
- What happened
- Steps to reproduce

### 💡 Suggest a Feature
Open an issue with `[Feature]` in the title. Tell us:
- What problem it solves
- Who it helps (builder? agent? enterprise?)

### 🔧 Submit Code
1. Fork the repo
2. Create a branch: `feature/your-idea` or `fix/your-bug`
3. Write code. Keep it simple.
4. Open a PR. Include what you changed and why.

### 📊 Become a Data Partner
If you run an agent framework or MCP gateway that generates real execution data, we want to work with you. Real execution data makes trust scores more accurate for everyone.

Open an issue with `[Partner]` in the title or contact us directly.

### 📝 Write About Us
Blog posts, tweets, conference talks — all help. Tag us or open an issue so we can amplify.

## Development

```bash
git clone https://github.com/agent-tool-intel/agent-tool-intel.git
cd agent-tool-intel
npm install
cp .env.example .env  # add your DATABASE_URL and DASHSCOPE_API_KEY
npm run dev
```

## What We Value

- **Transparency** — scores are explainable. Methodology is public.
- **Agent-first** — design for AI agents, not human dashboards.
- **Open data** — ecosystem stats belong to the community.
- **No spam** — we don't do cold outreach. Pull, never push.

## Code of Conduct

Be kind. Be helpful. Don't spam.
